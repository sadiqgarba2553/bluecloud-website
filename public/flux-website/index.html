Why

Why another language?

Python is too slow. Rust is too hard. Go is too limited.

Every language forces a tradeoff. Python gives you simplicity but takes your performance. Rust gives you speed but takes your afternoons fighting the borrow checker. Go gives you concurrency but takes your generics and expressiveness.

Flux doesn't ask you to choose. It starts simple — you write a script and run it. When you need speed, compile to native. When you need safety, the type system has your back. No tradeoffs.

One command. That's it.

No virtual environments. No package.json. No Makefile. No CMakeLists.txt. No gradle. No xcodeproj.

You write a .flux file and run it. The standard library includes system monitoring, HTTP servers, file I/O, and string manipulation out of the box. If Python taught us anything, it's that batteries should be included.

Flux is a language for people who want to build things, not configure build systems.
