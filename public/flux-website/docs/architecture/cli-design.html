Flux CLI Design

The flux command is the unified entry point for creating, building, testing, managing, and publishing Flux projects.

1. Command Reference

1.1 Project Management

Command Description
flux new <name> Create a new project
flux init Initialize Flux in current directory
flux build Compile the project
flux run Build and run the project
flux clean Remove build artifacts
flux check Type-check without generating code

1.2 Dependencies

Command Description
flux add <package> Add a dependency
flux remove <package> Remove a dependency
flux update Update dependencies
flux install Install all dependencies
flux tree Display dependency tree
flux audit Check dependencies for vulnerabilities

1.3 Code Quality

Command Description
flux format Format source files
flux lint Run the linter
flux test Run tests
flux bench Run benchmarks
flux doc Generate documentation

1.4 Publishing

Command Description
flux publish Publish to the package registry
flux login Authenticate with the registry
flux logout Clear authentication
flux search <query> Search the registry
flux info <package> Show package details

1.5 Tools

Command Description
flux doctor Diagnose installation issues
flux target list List available compilation targets
flux target add <target> Add a compilation target
flux self update Update Flux to the latest version
flux self uninstall Uninstall Flux
flux completions <shell> Generate shell completions

2. Detailed Command Specifications

flux new

flux new <name> [options]

Options:
  --lib           Create a library project
  --bin           Create a binary project (default)
  --edition <y>   Set the edition (default: 2026)
  --vcs <type>    Initialize version control (git, none) (default: git)
  --template <t>  Use a project template

Examples:
  flux new my_app
  flux new my_lib --lib
  flux new web_app --template web

flux build

flux build [options]

Options:
  --release          Build in release mode (optimized)
  --target <triple>  Cross-compile for target
  --jobs <N>         Number of parallel jobs
  --verbose          Show detailed build output
  --opt-level <N>    Optimization level (0, 1, 2, 3, s, z)
  --features <f>     Enable features (comma-separated)
  --all-features     Enable all features
  --no-default       Disable default features
  --timings          Show build timing breakdown

Examples:
  flux build
  flux build --release
  flux build --target aarch64-apple-darwin
  flux build --release --opt-level 3

flux test

flux test [filter] [options]

Options:
  --release          Run tests in release mode
  --lib              Test only library
  --doc              Run documentation tests
  --jobs <N>         Parallel test threads
  --no-capture       Show test output
  --coverage         Generate coverage report
  --filter <name>    Run tests matching name
  --ignored          Run ignored tests
  --timeout <secs>   Set test timeout

Examples:
  flux test
  flux test test_parsing
  flux test --coverage
  flux test --filter "user::*"

flux format

flux format [options] [files...]

Options:
  --check      Check formatting without modifying files
  --diff       Show formatting diff
  --config <f> Use custom format configuration

Examples:
  flux format
  flux format --check
  flux format src/main.flux

flux lint

flux lint [options]

Options:
  --fix          Auto-fix linting issues
  --strict       Enable all lint rules
  --config <f>   Use custom lint configuration
  --warn <rule>  Set specific rule to warning
  --deny <rule>  Set specific rule to error
  --allow <rule> Disable specific rule

Examples:
  flux lint
  flux lint --fix
  flux lint --deny unused-variables

3. Configuration

3.1 Flux.toml Build Configuration

[profile.dev]
opt_level = 0
debug = true
overflow_checks = true
incremental = true

[profile.release]
opt_level = 3
debug = false
lto = "thin"         # Link-time optimization: "off", "thin", "fat"
strip = true          # Strip debug symbols
codegen_units = 1     # Single codegen unit for max optimization
panic = "unwind"      # "unwind" or "abort"

[profile.bench]
inherits = "release"
debug = true          # Keep debug info for profiling

[build]
jobs = 8              # Parallel compilation jobs
target_dir = "target" # Build output directory

3.2 Environment Variables

Variable Description Default
FLUX_HOME Flux installation directory ~/.flux
FLUX_LOG Log level warn
FLUX_THREADS Compilation threads CPU count
FLUX_TARGET Default compilation target Host
FLUX_CACHE_DIR Cache directory ~/.flux/cache
FLUX_REGISTRY Package registry URL https://packages.flux-lang.org

4. Exit Codes

Code Meaning
0 Success
1 Compilation error
2 Usage error (bad arguments)
3 Test failure
4 Lint error
100 Internal compiler error (bug)
101 Panic
126 Command not executable
127 Command not found

5. Output Format

5.1 Colorized Diagnostics

The CLI uses color to improve readability: - Red: Errors - Yellow: Warnings - Blue: Notes and help - Green: Success messages - Cyan: File paths

Color can be disabled with --color never or NO_COLOR=1.

5.2 JSON Output

For tool integration, all commands support --output json:

flux build --output json
flux test --output json
flux lint --output json

{
  "diagnostics": [
    {
      "severity": "error",
      "code": "E0382",
      "message": "use of moved value: `name`",
      "file": "src/main.flux",
      "line": 5,
      "column": 11,
      "suggestions": [
        {
          "message": "consider cloning the value",
          "replacement": "name.clone()"
        }
      ]
    }
  ]
}

6. Shell Completions

# Bash
flux completions bash > /etc/bash_completion.d/flux

# Zsh
flux completions zsh > ~/.zfunc/_flux

# Fish
flux completions fish > ~/.config/fish/completions/flux.fish

# PowerShell
flux completions powershell > flux.ps1
