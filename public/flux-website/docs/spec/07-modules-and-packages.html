Flux Language Specification — Modules and Packages

Version: 0.1.0-draft
Status: Draft

1. Overview

Flux uses a hierarchical module system for code organization and a package system for dependency management. Modules provide namespacing and encapsulation. Packages are distributable units of code with versioned dependencies.

2. Modules

2.1 Module Declaration

Every .flux file implicitly defines a module. The module name is derived from the file name.

src/
├── main.flux          → root module
├── utils.flux         → module `utils`
├── models/
│   ├── mod.flux       → module `models`
│   ├── user.flux      → module `models.user`
│   └── post.flux      → module `models.post`
└── services/
    ├── mod.flux        → module `services`
    └── auth.flux       → module `services.auth`

2.2 Inline Modules

Modules can be declared inline within a file:

mod math {
    pub fn abs(x: Int) -> Int {
        if x < 0 { return -x }
        return x
    }

    pub fn clamp(value: Int, min: Int, max: Int) -> Int {
        if value < min { return min }
        if value > max { return max }
        return value
    }
}

// Usage
let result = math.abs(-42)

2.3 File Modules

Declare a module that loads from a separate file:

// In main.flux
mod utils       // Loads from utils.flux or utils/mod.flux
mod models      // Loads from models/mod.flux

fn main() {
    let user = models.user.User { name: "Alice" }
}

2.4 Module Documentation

//! This module provides HTTP client and server functionality.
//!
//! # Examples
//!
//! ```flux
//! use flux.http.{Client, Request}
//!
//! async fn fetch() {
//!     let client = Client.new()
//!     let response = await client.get("https://example.com")
//! }
//! ```

pub struct Client { ... }
pub struct Server { ... }

3. Visibility

3.1 Visibility Modifiers

Modifier Scope Description
(none) Private Only accessible within the current module
pub Public Accessible from anywhere
pub(crate) Crate-public Accessible within the current package
pub(super) Parent-public Accessible in the parent module
pub(mod) Module-public Accessible in the current module and submodules 
mod shapes {
    // Public — accessible everywhere
    pub struct Circle {
        pub radius: Float64           // Public field
        center: Point                 // Private field
    }

    // Crate-public — only within this package
    pub(crate) fn internal_helper() { ... }

    // Private — only within this module
    fn calculate_area(r: Float64) -> Float64 {
        return PI * r * r
    }
}

3.2 Default Visibility

  • Items (functions, types, constants) are private by default
  • Struct fields are private by default
  • Enum variants are public if the enum is public
  • Trait methods are public (traits define public contracts)
  • Impl methods follow their pub annotation

4. Imports (Use Declarations)

4.1 Basic Imports

use flux.io.print                        // Single item
use flux.collections.HashMap             // Type import
use flux.math.{sin, cos, tan}           // Multiple items
use flux.http.*                          // Glob import (discouraged)
use flux.json.parse as json_parse       // Aliased import

4.2 Nested Imports

use flux.collections.{
    HashMap
    HashSet
    TreeMap
    List.{sort, reverse}      // Nested
}

4.3 Re-exports

// In models/mod.flux
pub use self.user.User          // Re-export User from submodule
pub use self.post.Post
pub use self.comment.Comment

// Now external code can do:
use mypackage.models.User       // Instead of mypackage.models.user.User

4.4 Import Rules

  1. Imports are resolved at compile time
  2. Circular imports are allowed (the compiler handles them)
  3. Unused imports generate a warning
  4. The _ prefix suppresses unused import warnings: use flux.io._

5. Package System

5.1 Package Manifest (Flux.toml)

Every Flux package has a Flux.toml manifest file:

[package]
name = "my_app"
version = "1.2.3"
edition = "2026"
authors = ["Alice <alice@example.com>"]
description = "A sample Flux application"
license = "MIT"
repository = "https://github.com/alice/my_app"
homepage = "https://my-app.dev"
keywords = ["web", "server", "api"]
categories = ["web-programming"]

[dependencies]
flux_http = "2.1"
flux_json = "1.0"
flux_db = { version = "3.0", features = ["postgres", "mysql"] }
my_lib = { git = "https://github.com/alice/my_lib.git", branch = "main" }
local_lib = { path = "../local_lib" }

[dev-dependencies]
flux_testing = "1.0"
flux_mock = "0.5"

[build-dependencies]
flux_codegen = "1.0"

[features]
default = ["json", "yaml"]
json = ["flux_json"]
yaml = ["flux_yaml"]
full = ["json", "yaml", "xml", "csv"]

[profile.release]
opt_level = 3
lto = true
strip = true

[profile.dev]
opt_level = 0
debug = true

5.2 Version Constraints

Syntax Meaning
"1.2.3" Compatible with 1.2.3 (≥1.2.3, <2.0.0)
"=1.2.3" Exactly 1.2.3
">=1.2, <2.0" Range constraint
"~1.2.3" Patch-compatible (≥1.2.3, <1.3.0)
"^1.2.3" Compatible (same as "1.2.3")
"*" Any version

5.3 Lock File (Flux.lock)

The lock file records exact resolved versions for reproducible builds:

[[package]]
name = "flux_http"
version = "2.1.4"
source = "registry+https://packages.flux-lang.org"
checksum = "sha256:abc123..."

[[package]]
name = "flux_json"
version = "1.0.2"
source = "registry+https://packages.flux-lang.org"
dependencies = ["flux_io 1.0.1"]

5.4 Workspaces

Multi-package projects use workspaces:

# Root Flux.toml
[workspace]
members = [
    "compiler/flux_lexer",
    "compiler/flux_parser",
    "compiler/flux_ast",
    "cli",
    "stdlib",
]

[workspace.dependencies]
flux_ast = { path = "compiler/flux_ast" }

6. Conditional Compilation

6.1 Configuration Attributes

#[cfg(target_os: "linux")]
fn get_home_dir() -> String {
    return env("HOME")
}

#[cfg(target_os: "windows")]
fn get_home_dir() -> String {
    return env("USERPROFILE")
}

#[cfg(target_os: "macos")]
fn get_home_dir() -> String {
    return env("HOME")
}

6.2 Available Configuration Options

Option Values
target_os "linux", "macos", "windows", "android", "ios"
target_arch "x86_64", "aarch64", "arm", "wasm32"
target_family "unix", "windows", "wasm"
target_endian "little", "big"
target_pointer_width "32", "64"
debug true, false
test true (when running tests)
feature Any feature name from Flux.toml

6.3 Combining Conditions

#[cfg(all(target_os: "linux", target_arch: "x86_64"))]
fn optimized_linux_x64() { ... }

#[cfg(any(target_os: "linux", target_os: "macos"))]
fn unix_function() { ... }

#[cfg(not(target_os: "windows"))]
fn non_windows() { ... }

#[cfg(feature: "json")]
use flux.json

6.4 Feature Flags

// In library code
#[cfg(feature: "async")]
pub async fn fetch(url: String) -> Response { ... }

#[cfg(not(feature: "async"))]
pub fn fetch(url: String) -> Response { ... }

7. Build Scripts

Packages can include a build.flux script that runs before compilation:

// build.flux
use flux.build.{Config, generate_bindings}

fn main() {
    let config = Config.new()

    // Generate FFI bindings from a C header
    generate_bindings("vendor/sqlite3.h")
        .output("src/bindings.flux")
        .generate()

    // Link a native library
    config.link_lib("sqlite3")
    config.link_search("/usr/local/lib")

    // Set compile-time environment variables
    config.set_env("BUILD_TIMESTAMP", time.now().to_string())
}
