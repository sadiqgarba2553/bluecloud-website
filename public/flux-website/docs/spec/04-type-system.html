Flux Language Specification — Type System

Version: 0.1.0-draft
Status: Draft

1. Overview

Flux has a strong, static type system with extensive type inference. Types are checked at compile time, preventing entire categories of runtime errors. The type system is designed to be expressive enough for systems programming while remaining approachable for developers coming from dynamically typed languages.

Design goals: - Catch bugs at compile time, not at runtime - Minimize the need for explicit type annotations through inference - Support zero-cost abstractions via monomorphized generics - Enforce null safety and memory safety through the type system

2. Primitive Types

2.1 Integer Types

Int16 16-bit -32,768 to 32,767 Signed short
Int32 32-bit -(2³¹) to 2³¹-1 Signed 32-bit
Int64 64-bit -(2⁶³) to 2⁶³-1 Signed 64-bit
UInt8 8-bit 0 to 255 Unsigned byte
UInt16 16-bit 0 to 65,535 Unsigned short

UInt64 64-bit 0 to 2⁶⁴-1 Unsigned 64-bit

let a = 42            // Int
let b: Int8 = 42      // Int8
let c = 42u32         // UInt32 (suffix)
let d: USize = 1024   // Pointer-size unsigned

2.2 Floating-Point Types

Type Size Precision Description
Float32 32-bit ~7 decimal digits IEEE 754 single precision
Float64 64-bit ~15 decimal digits IEEE 754 double precision 
let pi = 3.14159           // Float64 (default)
let precise: Float64 = 1.0
let fast: Float32 = 1.0f32

2.3 Boolean Type

Type Size Values
Bool 1 byte true, false 
let enabled = true
let visible: Bool = false

2.4 Character Type

Type Size Description
Char 4 bytes Single Unicode scalar value (U+0000 to U+10FFFF, excluding surrogates) 
let letter = 'A'
let emoji = '🔥'
let tab = '\t'

2.5 String Type

String is a growable, heap-allocated, UTF-8 encoded text type.

let greeting = "Hello, World!"
let name = "Flux"
let interpolated = "Welcome to {name}!"

Strings are immutable value types with copy-on-write semantics. Modifications create a new string.

2.6 Byte Type

Type Size Description
Byte 1 byte Alias for UInt8, used for raw binary data 
let data: [Byte] = [0xFF, 0xAA, 0x00]

2.7 Void Type

Void represents the absence of a value. It is the return type of functions that do not return anything.

fn log(message: String) {   // Return type is implicitly Void
    print(message)
}

2.8 Never Type

Never (also written !) represents a type with no values. It is used for functions that never return (e.g., infinite loops, panics).

fn crash(message: String) -> Never {
    panic(message)
}

3. Compound Types

3.1 Tuples

Tuples are fixed-size, heterogeneous collections of values.

let point: (Int, Int) = (10, 20)
let record: (String, Int, Bool) = ("Alice", 30, true)

// Access by index
print(point.0)   // 10
print(point.1)   // 20

// Named tuples
let color = (r: 255, g: 128, b: 0)
print(color.r)   // 255

// Destructuring
let (x, y) = point
let (name, age, _) = record

// Unit tuple (empty)
let unit: () = ()

3.2 Arrays

Arrays are fixed-size, contiguous sequences of values of the same type. Size is part of the type.

let numbers: [Int; 5] = [1, 2, 3, 4, 5]
let zeros = [0; 100]   // 100 zeros

// Size is known at compile time
print(numbers.length)   // 5

3.3 Slices

Slices are dynamically-sized views into a contiguous sequence. They are references — they borrow from an array or other collection.

let numbers = [1, 2, 3, 4, 5]
let middle: &[Int] = &numbers[1..4]   // [2, 3, 4]

3.4 Lists

List<T> is a growable, heap-allocated sequence.

let names: List<String> = ["Alice", "Bob", "Carol"]
mut items = List<Int>.new()
items.append(1)
items.append(2)
print(items.length)   // 2

3.5 Maps

HashMap<K, V> and TreeMap<K, V> are key-value associative containers.

let scores: HashMap<String, Int> = {
    "Alice": 95
    "Bob": 87
}

// Access
print(scores["Alice"])   // Option<Int> — returns Some(95)

3.6 Sets

HashSet<T> and TreeSet<T> are collections of unique values.

let colors = Set{"red", "green", "blue"}
print(colors.contains("red"))   // true

4. Option and Result Types

4.1 Option\<T>

Option<T> represents a value that may or may not exist. It replaces null/nil entirely.

enum Option<T> {
    Some(T),
    None,
}

fn find_user(id: Int) -> Option<User> {
    if id == 1 {
        return Some(User { name: "Alice" })
    }
    return None
}

// Usage
let user = find_user(1)

// Pattern matching
match user {
    Some(u) -> print("Found: {u.name}")
    None    -> print("Not found")
}

// Shorthand syntax
let user: User? = find_user(1)   // User? is sugar for Option<User>

// Optional chaining
let name = user?.name ?? "Anonymous"

// If-let unwrapping
if let Some(u) = find_user(1) {
    print("Found: {u.name}")
}

// Force unwrap (panics on None — use sparingly)
let u = find_user(1).unwrap()

// Map and flat_map
let upper_name = find_user(1).map(fn(u) { u.name.to_upper() })

4.2 Result\<T, E>

Result<T, E> represents an operation that may succeed or fail with an error.

enum Result<T, E> {
    Ok(T),
    Err(E),
}

fn divide(a: Int, b: Int) -> Result<Int, MathError> {
    if b == 0 {
        return Err(MathError.DivisionByZero)
    }
    return Ok(a / b)
}

// Error propagation
fn compute() -> Result<Int, MathError> {
    let x = divide(10, 2)?   // Unwraps Ok or returns Err
    let y = divide(x, 3)?
    return Ok(x + y)
}

5. Type Inference

Flux uses a Hindley-Milner-inspired type inference algorithm extended with trait bounds. The compiler infers types from:

  1. Literal types: 42 → Int, 3.14 → Float64, "hello" → String
  2. Variable initialization: let x = expression infers x's type from expression
  3. Function return: The return type can often be inferred (but must be explicitly declared for public functions)
  4. Generic instantiation: List.new() infers T from context
  5. Closure parameters: items.map((x) => x * 2) infers x's type from items's element type

5.1 Where Inference Works

let x = 42                           // Int
let name = "Flux"                    // String
let items = [1, 2, 3]               // List<Int>
let map = {"a": 1, "b": 2}          // HashMap<String, Int>
let result = add(1, 2)              // Inferred from add's return type
let doubled = items.map((x) => x * 2) // Inferred from List<Int>

// Type flows through assignments
let a = 42
let b = a       // Int (inferred from a)
let c = b + 1   // Int (inferred from + operator)

5.2 Where Annotations Are Required

// Public function signatures (always required)
pub fn add(a: Int, b: Int) -> Int {
    return a + b
}

// Struct/class/enum fields (always required)
struct Point {
    x: Float64
    y: Float64
}

// Empty collections (type cannot be inferred)
let items: List<Int> = List.new()
let map: HashMap<String, Int> = HashMap.new()

// Ambiguous numeric contexts
let x: Float32 = 1.0   // Could be Float32 or Float64

5.3 Bidirectional Type Inference

Type information flows both forward and backward:

// Forward: type of `items` determines type of `x` in the closure
let items: List<Int> = [1, 2, 3]
let doubled = items.map((x) => x * 2)  // x is Int

// Backward: expected type of `result` determines generic parameter
let result: List<String> = items.map((x) => x.to_string())

// Both directions:
let callback: fn(Int) -> Bool = (x) => x > 0

6. Generics

6.1 Generic Functions

fn identity<T>(value: T) -> T {
    return value
}

fn swap<A, B>(pair: (A, B)) -> (B, A) {
    return (pair.1, pair.0)
}

6.2 Generic Types

struct Pair<A, B> {
    first: A
    second: B
}

enum Tree<T> {
    Leaf(T),
    Node(left: Box<Tree<T>>, right: Box<Tree<T>>),
}

6.3 Trait Bounds

fn largest<T: Comparable>(list: List<T>) -> T {
    mut max = list[0]
    for item in list[1..] {
        if item.greater_than(max) {
            max = item
        }
    }
    return max
}

// Multiple bounds
fn display<T: Printable + Comparable>(item: T) {
    print(item.to_string())
}

// Where clause for complex bounds
fn merge_maps<K, V>(a: HashMap<K, V>, b: HashMap<K, V>) -> HashMap<K, V>
    where K: Hashable + Eq,
          V: Clone {
    mut result = a.clone()
    for (key, value) in b {
        result.insert(key, value.clone())
    }
    return result
}

6.4 Monomorphization

Generics are monomorphized at compile time. List<Int> and List<String> generate separate, specialized code. This provides: - Zero runtime overhead (no boxing, no virtual dispatch) - Optimal performance (LLVM can optimize each specialization) - Larger binary size (trade-off)

// The compiler generates specialized versions:
// identity_Int(value: Int) -> Int
// identity_String(value: String) -> String
let a = identity(42)        // Uses identity<Int>
let b = identity("hello")   // Uses identity<String>

6.5 Associated Types

trait Collection {
    type Item
    type Iterator: Iterator<Item = Self.Item>

    fn iter(self) -> Self.Iterator
    fn length(self) -> Int
}

impl Collection for List<Int> {
    type Item = Int
    type Iterator = ListIterator<Int>

    fn iter(self) -> ListIterator<Int> {
        return ListIterator.new(self)
    }

    fn length(self) -> Int {
        return self.len
    }
}

7. Subtyping and Variance

7.1 Nominal Subtyping

Flux uses nominal subtyping for classes (through inheritance) and structural subtyping for interfaces.

class Animal { ... }
class Dog: Animal { ... }

// Dog is a subtype of Animal
let animal: Animal = Dog(name: "Rex", breed: "Lab")

7.2 Structural Subtyping (Interfaces)

A type satisfies an interface if it has all the required methods, even without an explicit impl declaration:

interface Stringable {
    fn to_string(self) -> String
}

struct Point {
    x: Float64
    y: Float64
}

impl Point {
    fn to_string(self) -> String {
        return "({self.x}, {self.y})"
    }
}

// Point satisfies Stringable without explicit declaration
fn print_item(item: Stringable) {
    print(item.to_string())
}

print_item(Point { x: 1.0, y: 2.0 })  // Works!

7.3 Variance

Container T Position Variance Meaning
List<T> Output Covariant List<Dog> is subtype of List<Animal> (read-only)
fn(T) Input Contravariant fn(Animal) is subtype of fn(Dog)
Cell<T> Both Invariant Cell<Dog> is NOT subtype of Cell<Animal>
&T Output Covariant &Dog is subtype of &Animal
&mut T Both Invariant &mut Dog is NOT subtype of &mut Animal

8. Type Aliases

type StringMap<V> = HashMap<String, V>
type Callback = fn(Int, Int) -> Int
type AppResult<T> = Result<T, AppError>
type Coordinates = (Float64, Float64)
type Matrix = List<List<Float64>>

9. Type Conversions

9.1 Implicit Conversions

Flux performs very few implicit conversions to prevent subtle bugs:

From To Rule
Int8 → Int16 → Int32 → Int64 Widening Always safe
UInt8 → UInt16 → UInt32 → UInt64 Widening Always safe
Float32 → Float64 Widening Always safe
Integer → Float Widening When no precision loss 
let x: Int8 = 42
let y: Int64 = x   // OK: implicit widening

9.2 Explicit Conversions (as)

let x: Int = 256
let y: Int8 = x as Int8     // Truncation (wraps to 0)
let z: Float64 = x as Float64
let w: Int = 3.14 as Int    // Truncation (3)

9.3 Safe Conversions (try_into)

let big: Int64 = 99999
let result: Result<Int8, OverflowError> = big.try_into()
match result {
    Ok(value) -> print("Fits: {value}")
    Err(_)    -> print("Too big for Int8")
}

9.4 Custom Conversions

trait From<T> {
    fn from(value: T) -> Self
}

trait Into<T> {
    fn into(self) -> T
}

// Implementing From automatically provides Into
impl From<String> for UserId {
    fn from(value: String) -> UserId {
        return UserId { id: value }
    }
}

let id: UserId = UserId.from("user_123")
let id2: UserId = "user_123".into()   // Works via Into

10. Dynamic Typing (any)

For scripting and interop scenarios, Flux provides an any type that opts into dynamic typing:

let value: any = 42
let name: any = "Flux"

// Type checking at runtime
if value is Int {
    let n = value as Int
    print("Integer: {n}")
}

// Dynamic dispatch
fn process(data: any) {
    match data {
        is Int    -> print("Int: {data as Int}")
        is String -> print("String: {data as String}")
        is List<any> -> print("List with {(data as List<any>).length} items")
        _ -> print("Unknown type")
    }
}

Note: Using any disables compile-time type checking for those values. The linter warns on any usage and suggests typed alternatives.

11. Function Types

Functions are first-class values with explicit types:

// Function type syntax: fn(ParamTypes...) -> ReturnType
type BinaryOp = fn(Int, Int) -> Int
type Predicate<T> = fn(T) -> Bool
type Action = fn() -> Void

let add: fn(Int, Int) -> Int = fn(a, b) { return a + b }
let is_even: fn(Int) -> Bool = (n) => n % 2 == 0

// Higher-order functions
fn apply(f: fn(Int) -> Int, value: Int) -> Int {
    return f(value)
}

let result = apply((x) => x * 2, 21)  // 42

12. Phantom Types

Phantom types carry type-level information without any runtime representation:

struct Meters {}
struct Seconds {}

struct Quantity<Unit> {
    value: Float64
    _phantom: PhantomData<Unit>
}

fn add_quantities<U>(a: Quantity<U>, b: Quantity<U>) -> Quantity<U> {
    return Quantity { value: a.value + b.value, _phantom: PhantomData }
}

let distance = Quantity<Meters> { value: 100.0, _phantom: PhantomData }
let time = Quantity<Seconds> { value: 9.58, _phantom: PhantomData }

// add_quantities(distance, time)  // Compile error: Meters ≠ Seconds
add_quantities(distance, distance)  // OK

13. Recursive Types

Recursive types are supported through indirection (Box<T>):

enum JsonValue {
    Null,
    Bool(Bool),
    Number(Float64),
    Str(String),
    Array(List<JsonValue>),
    Object(HashMap<String, JsonValue>),
}

// Binary tree
enum Tree<T> {
    Leaf(T),
    Node {
        value: T
        left: Box<Tree<T>>
        right: Box<Tree<T>>
    },
}

15. Type Safety Guarantees

The Flux type system provides the following guarantees:

  1. No null pointer dereferences — All nullable values are Option<T> and must be explicitly unwrapped
  2. No uninitialized variables — All variables must be initialized before use
  3. No implicit type coercions (except safe widening) — Explicit conversion required
  4. No data races — Send and Sync traits enforce thread safety at compile time
  5. Exhaustive matching — match expressions must cover all possible cases
  6. No use-after-move — The compiler tracks ownership and prevents use of moved values
  7. No implicit lossy conversions — Narrowing conversions require explicit as or try_into()
