Flux Language Specification — Metaprogramming

Version: 0.1.0-draft
Status: Draft

1. Overview

Flux provides metaprogramming capabilities through compile-time evaluation, macros, attributes, and limited reflection. These features enable code generation, domain-specific languages, and compile-time optimization without sacrificing type safety.

2. Compile-Time Evaluation (comptime)

2.1 Comptime Blocks

comptime blocks execute during compilation. They can compute values, generate types, and perform static assertions:

const TABLE_SIZE = comptime {
    let mut size = 1
    while size < 1000 {
        size *= 2
    }
    size   // Result: 1024
}

// Compile-time string processing
const VERSION_MAJOR = comptime {
    let version = env("CARGO_PKG_VERSION")
    let parts = version.split(".")
    parts[0].parse_int().unwrap()
}

2.2 Comptime Functions

Functions marked with comptime can be called at compile time:

comptime fn fibonacci(n: Int) -> Int {
    if n <= 1 { return n }
    return fibonacci(n - 1) + fibonacci(n - 2)
}

const FIB_10 = fibonacci(10)   // Computed at compile time: 55

// Comptime type generation
comptime fn make_vector_type(n: Int) -> Type {
    // Returns a struct with n Float64 fields
    // This is a simplified example of the API
    let fields = (0..n).map(fn(i) {
        Field { name: "v{i}", type: Float64 }
    })
    return Struct { name: "Vec{n}", fields: fields }
}

2.3 Static Assertions

const _: () = comptime {
    assert(size_of<Int>() >= 4, "Int must be at least 32 bits")
    assert(align_of<Pointer>() == 8, "Pointers must be 8-byte aligned")
}

// Shorthand
static_assert(size_of<Int>() == 8)
static_assert(MAX_BUFFER_SIZE > 0, "Buffer size must be positive")

3. Macros

3.1 Declarative Macros

Declarative macros use pattern matching on syntax to expand code:

macro vec!($($elem:expr),* $(,)?) {
    {
        let mut v = List.new()
        $(v.append($elem))*
        v
    }
}

// Usage
let numbers = vec![1, 2, 3, 4, 5]
let names = vec!["Alice", "Bob", "Carol"]

macro hashmap!($($key:expr => $value:expr),* $(,)?) {
    {
        let mut map = HashMap.new()
        $(map.insert($key, $value))*
        map
    }
}

let scores = hashmap!{
    "Alice" => 95,
    "Bob" => 87,
    "Carol" => 92,
}

3.2 Macro Pattern Syntax

Pattern Matches
$name:expr Any expression
$name:ident An identifier
$name:type A type annotation
$name:pat A pattern
$name:stmt A statement
$name:block A block { ... }
$name:literal A literal value
$name:tt A single token tree
$($pattern)* Zero or more repetitions
$($pattern)+ One or more repetitions
$($pattern),* Comma-separated repetitions

3.3 Procedural Macros

Procedural macros are functions that receive a token stream and produce a token stream. They are more powerful than declarative macros.

Derive Macros

// Defining a derive macro (in a separate crate)
#[proc_macro_derive(Serialize)]
fn derive_serialize(input: TokenStream) -> TokenStream {
    let struct_def = parse_struct(input)

    let fields = struct_def.fields.map(fn(field) {
        quote! {
            map.insert("{field.name}", self.{field.name}.serialize())
        }
    })

    return quote! {
        impl Serialize for {struct_def.name} {
            fn serialize(self) -> JsonValue {
                let mut map = HashMap.new()
                {fields}
                return JsonValue.Object(map)
            }
        }
    }
}

// Usage
#[derive(Serialize)]
struct User {
    name: String
    age: Int
    email: String
}

Attribute Macros

// Defining an attribute macro
#[proc_macro_attribute]
fn route(attr: TokenStream, item: TokenStream) -> TokenStream {
    let method_and_path = parse_route_attr(attr)
    let function = parse_function(item)

    return quote! {
        #[http_route(method: {method_and_path.method}, path: {method_and_path.path})]
        {function}
    }
}

// Usage
#[route(GET, "/api/users")]
fn list_users(req: Request) -> Response {
    // ...
}

#[route(POST, "/api/users")]
fn create_user(req: Request) -> Response {
    // ...
}

Function-Like Macros

// Defining a function-like macro
#[proc_macro]
fn sql(input: TokenStream) -> TokenStream {
    let query = parse_sql(input)
    validate_sql_syntax(query)

    return quote! {
        SqlQuery {
            text: {query.text}
            params: vec![{query.params}]
        }
    }
}

// Usage — compile-time SQL validation
let query = sql!(SELECT name, email FROM users WHERE age > 18)
let results = db.execute(query)

5. Reflection

Flux provides limited, opt-in reflection for types that derive Reflect:

#[derive(Reflect)]
struct User {
    name: String
    age: Int
    email: String
}

fn inspect(value: &dyn Reflect) {
    let type_info = value.type_info()
    print("Type: {type_info.name}")
    print("Fields: {type_info.field_count}")

    for field in type_info.fields() {
        print("  {field.name}: {field.type_name} = {field.get(value)}")
    }
}

let user = User { name: "Alice", age: 30, email: "alice@flux.dev" }
inspect(&user)
// Output:
// Type: User
// Fields: 3
//   name: String = Alice
//   age: Int = 30
//   email: String = alice@flux.dev

5.1 Reflection API

trait Reflect {
    fn type_info(self) -> TypeInfo
    fn field_value(self, name: String) -> Option<any>
    fn set_field_value(mut self, name: String, value: any) -> Result<Void, ReflectError>
}

struct TypeInfo {
    name: String
    module: String
    kind: TypeKind  // Struct, Class, Enum, etc.

    fn fields(self) -> List<FieldInfo>
    fn methods(self) -> List<MethodInfo>
}

struct FieldInfo {
    name: String
    type_name: String
    is_public: Bool
    attributes: List<AttributeInfo>
}

5.2 Reflection Limitations

Reflection is intentionally limited for performance:

  1. Only available for types that opt in with #[derive(Reflect)]
  2. No arbitrary method invocation through reflection
  3. No runtime type creation
  4. No reflection on private fields (unless explicitly opted in)
  5. Reflection data is stripped in release builds by default (can be kept with #[cfg(reflect)])

6.1 Build-Time Code Generation

use flux.build.codegen

fn main() {
    // Generate Flux code from a schema file
    codegen.from_file("schema.json")
        .template("templates/model.flux.tmpl")
        .output("src/generated/models.flux")
        .generate()
}

6.2 Include Macros

// Include file contents as a string at compile time
const SHADER_SOURCE = include_str!("shaders/vertex.glsl")

// Include file contents as bytes at compile time
const ICON_DATA = include_bytes!("assets/icon.png")

// Include and parse a file at compile time
const CONFIG = include_toml!("default_config.toml")

7. Annotations for Tooling

7.1 Linter Annotations

#[allow(unused_variables)]
fn experimental() {
    let x = 42   // No warning
}

#[deny(implicit_return)]
fn strict_function() -> Int {
    return 42   // Must use explicit return
}

// Suppress specific warnings for a scope
fn legacy_code() {
    #[allow(deprecated)]
    old_api_call()
}

7.2 Documentation Annotations

///
/// # Type Parameters
/// - `K` — The key type, must implement `Hash + Eq`
/// - `V` — The value type

/// # Examples

/// assert(map.get("key") == Some(42))

/// - Insert: O(1) amortized
/// - Lookup: O(1) average
/// - Delete: O(1) average
///
